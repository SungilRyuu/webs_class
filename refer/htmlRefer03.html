시맨틱 태그

HTML5에 도입된 시맨틱 태그(semantic tag)는 개발자와 브라우저에 의미있는 태그를 제공하게 됩니다.
HTML 이전에는 <div> 태그만 보고는 이 태그 안에 들어갈 내용의 의미를 알 수 없습니다. HTML5가 도입된 이후, 대략적으로 태그 안의 내용의 의미를 유추할 수 있게 되었습니다. 예를 들어, <table> 태그 안에는 표가 들어갈 것이고, <article> 태그 안에는 글이 들어갈 것이라는 것을 유추할 수 있습니다. 즉, 시맨틱 태그를 이용하여 HTML 태그의 의미를 명확하게 할 수 있습니다.

속성 설명
<section> <section> 태그는 주제별 그룹의 콘텐츠 섹션을 나타낼 때 사용합니다.
<header> <header>은 머리말 영역으로, 문서 또는 섹션의 헤더를 나타냅니다.
<nav> <nav>은 메뉴 영역으로, 다른 곳으로 이동할 수 있는 링크를 주로 표시합니다.
<main> <main>은 주요 콘텐츠를 나타냅니다.
한 페이지에 한 번만 사용 가능합니다.
<section> <section>은 어느 한 구역을 의미합니다.
제목을 가지고 있으면서 문서 전체와 관련이 있는 콘텐츠의 집합이라 볼 수 있습니다.
<article> <article>은 문서 본문 중 독립된 콘텐츠입니다.
<aside> <aside>는 사이드바라 불리며, 광고 영역 등으로 활용됩니다.
<footer> <footer>은 꼬리말 혹은 바닥글을 의미합니다.
주로 화면 제일 아래 위치하며, 회사소개, 약관, 저작권, 제작 정보 등을 표시합니다.

HTML5에서 의미가 변경된 태그

HTML5에서 의미가 변경된 태그입니다.

속성 설명
<a> <a>은 href 속성 없이 사용할 수 있게 되었습니다.
<acronym> <acronym>은 <abbr>로 대체되었습니다.
<adress> <adress>은 실제 우편물 주소를 표시할 때 사용하는 태그로 변경되었습니다.
<applet> <applet>은 <embed>로 대체되었습니다.
<b> <b>은 기존 텍스트를 진하게 표시할 뿐만 아니라, 제품 소개서 안에 들어가는 제품명이나 요약문서 안의 키워드처럼 특별하게 중요하지는 않지만 표시하려고 할 때 사용합니다.
<dir> <dir>은 <ul>로 대체되었습니다.
<hr> <hr>은 단락 단위로 주제를 바꾸고자 할 때 사용하도록 정의되었습니다.
<i> <i>은 기울임체 적용 뿐만 아니라 중요한 정보가 들어 있지 않은 주변 콘텐츠로부터 텍스트를 따로 표시할 때 사용하도록 정의되었습니다.
<menu> <menu>은 컨텐츠의 메뉴, 도구 모음 및 양식 컨트롤과 같이 실제 문서 메뉴 정보를 제공하는데 사용되도록 의미가 변경되었습니다.
<small> <small>은 세부 주석이나, 법적 인쇄문서의 작은 인쇄 정보를 표시할 때 사용되도록 의미가 변경되었습니다.
<strike> <strike>은 <del>과 <s>로 대체되었습니다.
<strong> <strong>은 단순히 진하게 표시하는 것보다 중요한 정보를 표시할 때 사용해야 한다고 정의되었습니다.
